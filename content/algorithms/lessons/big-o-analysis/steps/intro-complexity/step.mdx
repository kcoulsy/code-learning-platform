# Introduction to Complexity

When we analyze algorithms, we care about how their resource usage grows as input size increases. This is called **algorithmic complexity**.

## Why Complexity Matters

```c
// O(1) - Constant time
int getFirst(int arr[]) {
    return arr[0];
}

// O(n) - Linear time
int findMax(int arr[], int n) {
    int max = arr[0];
    for (int i = 1; i < n; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
    }
    return max;
}

// O(nÂ²) - Quadratic time
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```

## Key Concepts

1. **Input Size (n)**: The number of elements or size of the problem
2. **Growth Rate**: How operations increase as n grows
3. **Worst Case**: Maximum resources needed
4. **Asymptotic Behavior**: Performance as n approaches infinity

## Types of Complexity

- **Time Complexity**: How execution time grows
- **Space Complexity**: How memory usage grows
